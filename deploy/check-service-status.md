# 🔍 检查服务状态

## 请在宝塔面板操作：

### 方法 1：查看项目日志

1. 在 Node 项目列表找到 **sm-api-v2**
2. 点击右侧的 **"日志"** 按钮
3. 查看是否有错误信息

### 方法 2：在宝塔终端查看日志

```bash
# 查看PM2日志
pm2 logs sm-api-v2 --lines 50

# 或者查看错误日志
pm2 logs sm-api-v2 --err --lines 50
```

### 方法 3：检查端口占用

```bash
# 检查5000端口是否真的在监听
netstat -tulpn | grep 5000

# 或使用ss命令
ss -tulpn | grep 5000
```

### 方法 4：手动测试启动

```bash
cd /www/wwwroot/sm-api-v2
node dist/main.js
```

---

## 🎯 常见问题

### 问题 1：端口被占用

如果 5000 端口已被占用，需要修改端口或停止占用端口的进程

### 问题 2：数据库连接失败

检查 `prisma/dev.db` 文件是否存在且有权限

### 问题 3：环境变量问题

NestJS 可能需要某些环境变量
